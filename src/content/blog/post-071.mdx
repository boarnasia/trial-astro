title: 'TypeScript Discriminated Unions'
description: 'Learn TypeScript discriminated unions to model state machines and handle complex conditional logic safely.'
pubDate: 'Jun 09 2021'
---

TypeScript discriminated unions are a pattern for representing a value that can be one of several types, where each type has a unique identifying property — the discriminant. TypeScript narrows the type based on that property.

A classic example is a loading state type with three variants: loading, success with data, and error with a message. Using a status string as the discriminant, TypeScript knows exactly which properties are available after a check like if (state.status === 'success').

The exhaustiveness check is a powerful technique using discriminated unions. By handling all variants in a switch statement and adding a never check in the default branch, TypeScript will produce a compile error if a new variant is added without updating the handling code.
