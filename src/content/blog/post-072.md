---
title: 'Writing Documentation That Developers Love'
description: 'Tips and frameworks for writing technical documentation that developers actually read and find useful.'
pubDate: 'Jul 17 2021'
---

Great documentation is written for the reader, not the author. The key insight is that readers have different needs at different moments — tutorial, how-to guide, reference, and explanation — and each type of documentation serves a different need.

The Diátaxis framework categorizes documentation into four types based on when and why it is useful. Tutorials teach by doing, for learning. How-to guides solve specific problems, for practical use. Reference material describes the system accurately, for consultation. Explanations provide understanding, for reflection.

Code examples are the most important part of technical documentation. Show runnable, tested examples that reflect real-world usage. Examples that do not work, or that require significant modification to be useful, erode trust in the documentation overall.
