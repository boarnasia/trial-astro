TypeScript's standard library includes dozens of utility types for transforming and combining existing types. These types are building blocks for advanced type programming and commonly appear in library type definitions.

Partial makes all properties of a type optional — useful for update functions where only some properties are provided. Required is the opposite, making all properties required. Readonly prevents all properties from being reassigned.

Pick creates a new type with only specified properties from an existing type, while Omit excludes specified properties. Record creates a type with specific keys and a consistent value type. ReturnType extracts the return type of a function, which is invaluable when working with inferred types.
