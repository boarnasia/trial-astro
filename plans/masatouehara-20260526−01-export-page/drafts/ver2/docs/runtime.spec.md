# 動作要件

ビルド成果物 `exports/` の動作要件と検証方法。

## 1. 起動

```
cd exports && bun serve .
```

- 静的ファイルサーバとして起動できること
- 追加のビルド手順なしでブラウザから閲覧できること

## 2. 視覚的一致

- 元の top ページ（`bun run dev` での表示）とスクリーンショットが一致すること
- 出力には header / footer が含まれていて良い

## 3. ページ構成

- 現状の対象は top ページのみ（`/index.html`）
- 多ページ展開は本フェーズでは非対象

## 4. 検証手順（想定）

1. `bun run build:export --version-id=<id>` を実行
2. `cd exports && bun serve .` で起動
3. 元 dev サーバと並べて、スクリーンショット比較
4. ネットワークタブで 404 や CORS エラーが出ていないことを確認

## 5. 非対象 / 今後の検討

- スクリーンショット比較の自動化（手動確認前提）
- Lighthouse 等のパフォーマンス計測
- アクセシビリティ検証
