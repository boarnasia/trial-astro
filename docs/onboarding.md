# オンボーディング

## 概要

このリポジトリは Astro ベースのブログプロジェクトです。

- フレームワーク: Astro
- UI: React
- スタイル: SCSS + BEM + FLOCSS
- Headless UI: Radix UI
- コンポーネント開発: Storybook

主な設計・実装ルールは [feontend-design.md](./feontend-design.md) を参照してください。

## 主要ディレクトリ

- src/pages/: ルーティングとページエントリ
- src/components/: UI コンポーネント
- src/content/blog/: ブログ記事
- scripts/: 記事生成などの補助スクリプト
- tasks/: 作業メモと実装計画

## 開発 URL

開発サーバーの起動・停止は人間が管理します。

- Astro: http://localhost:4321/
- Storybook: http://localhost:6006/

変更確認は上記 URL を基準に行います。

## AI との役割分担

### 人間の役割

- 目的と完了条件を決める
- サーバーの起動と停止を管理する
- 画面や動作の最終確認を行う
- プロダクトとデザインの最終判断を行う

### AI の役割

- 影響範囲を調査する
- 小さく明確な変更を提案・実装する
- 差分や判断理由を整理する
- ドキュメントやタスクを整える

## 推奨ワークフロー

1. 目的と完了条件を先に決める
2. 関連する画面・ファイル・ドキュメントを AI に渡す
3. 小さくレビューしやすい単位で変更する
4. Astro または Storybook で確認する
5. 継続的な判断は docs や tasks に残す

## メモ

- 大きな一括変更より、小さな変更を優先する
- 文脈は tasks 配下に残して認識差分を減らす
- ルートの README は短く保ち、詳細は docs に分離する
